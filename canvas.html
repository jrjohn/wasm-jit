Each component's script is its own unique source (a template × constants baked from the index = simulating "AI generates a behavior per component"), each compiled to its own WASM module (may only call granted capabilities: sin, cos, out — the import table is the capability list). kernel weight = number of while substeps (simulating physics/geometry iteration). Both modes eat the same batch of script sources. The reason to pick WASM is the capability sandbox (each cell can only sin/cos/out, can't even call fetch); JS ties on speed but has zero isolation. This PoC does no fuel metering (production needs Worker+terminate or back-edge counting).
