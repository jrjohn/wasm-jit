# wasm-jit 的華嚴讀法 —— 這引擎的「理」

> 定本 · 2026-07-21
>
> 一份把華嚴宗的 **四法界 → 六相圓融 → 十玄門** 逐條對映到 wasm-jit 實際機制(附行號)的文件。
>
> **先講清楚立場(誠實,不灌迷湯)**:這是一個**結構同構**,不是說這段程式碼開悟了、也不宣稱任何果地上的意義。它之所以扣得上,是因為華嚴本就是一套嚴謹的「部分與整體、一與多如何互攝」的關係理論,而「**一支純函數 + 一張受能力圍籬的詞彙表**」恰恰也是一個嚴謹的部分/整體、一/多系統。**兩邊在講同一個關係結構**——同構因此而生,不是神秘巧合。這份文件的用途,是把這個關係結構講到最清楚,當作引擎的「理」的定本。

## 總綱:理、事、法界

| 華嚴 | wasm-jit | 在哪 |
|---|---|---|
| **理**(法·自性·無相) | 一支 cell——純函數,無位、無相、不佔一個像素,只是「給相位與身分,吐座標」 | `apps/reservoirs-native.html:78`(行為 cell 全文 `:78`–`:153`) |
| **事**(相·現象) | 畫布上此刻這 1320 顆彩點,及其聚成的長條/圓餅/折線 | `apps/reservoirs-native.html:205`(啞畫筆 `frame()`,畫點於 `:216`–`:219`) |
| **法界**(自足圓滿的世界) | 那張 import table——cell 能碰的一切,就這 6 個函式(`sin cos get set ld sd`),多一個都碰不到 | `src/lib.rs:93`(`UI_IMPORTS`);server 端把它變真:`loan-server/src/main.rs:50`(`fenced_linker`) |

核心的一句:**cell 是理,畫布是事,import table 是法界。** 三者的關係,就是底下逐條要拆的。

## 一、四法界

### 事法界(現象界)
畫布上一顆一顆有位置、有邊界、看得見的點。啞畫筆每幀把 cell 吐的座標畫成點:`apps/reservoirs-native.html:216`–`:219`。

### 理法界(本體界)
那支 cell 本身:讀相位 `prog` 與粒子身分,依 `axis` 吐 `x` / `y` / 凝聚度。無相之法。
- cell 讀入(相位+身分):`apps/reservoirs-native.html:79`
- cell 吐出(依 axis 選通道):`apps/reservoirs-native.html:152`

### 理事無礙(法相之間無中間層)
**算一次理,事就現前——中間沒有保留式 DOM、沒有 framework 的協調層在擋。** 啞畫筆是能想像的最薄一層膜:跟 cell 要點、畫點,不增不減、不擋不礙。
- 「跟 cell 要點」:`apps/reservoirs-native.html:216`(`cellCoord(p,0)` / `cellCoord(p,1)`)
- 「畫點」:`apps/reservoirs-native.html:218`
- 這層膜之外 cell 碰不到任何東西(理不入生滅之外)——圍籬:`src/lib.rs:93`

### 事事無礙(相與相之間本自互通)
關鍵在**凝聚度**這一維(「多出來的那一維」):
- `coh = 1 − 0.92·sin(π·prog)`:兩端=1(界線分明),中段→0(界線消融):`apps/reservoirs-native.html:143`
- 界線消融時,粒子四散互融——`scat = (1−coh)·46`:`apps/reservoirs-native.html:144`
- **morph 不是動畫特效,是把「相與相本來就通」顯現出來**:一顆點同時含著它的長條相/圓餅相/折線相(見下「隱顯俱成」)。

## 二、六相圓融(三對)

法藏《華嚴一乘教義分齊章》以「舍(房子)」喻六相:總相是舍,別相是梁柱,同相是共成一舍,異相是梁柱各別,成相是合而成舍,壞相是各住自位不失自性。對到這支 cell:

| 相 | 意 | wasm-jit | 行號 |
|---|---|---|---|
| **總相** | 整體為一 | 整張圖(長條/圓餅/折線,一個 gestalt) | `apps/reservoirs-native.html:205`(一幀=一圖) |
| **別相** | 部分各別 | 每一顆粒子 / 每一叢水庫 | `apps/reservoirs-native.html:215`(逐粒);身分 `:165`–`:170` |
| **同相** | 共成一體 | 全由**同一支 cell** 驅動、同赴一個 `prog` | cell `:78`;共用相位 `:210` |
| **異相** | 各持其性 | 每顆各帶 innate 抖動(`pu/pRect/pWedge/curveAmp`)+ 水庫色與值 | `apps/reservoirs-native.html:165`–`:170`, `:186`–`:187` |
| **成相** | 合而成形 | 凝聚度→1,眾粒成就整圖(各就其版面座標) | `coh` `:143`;版面座標 `:112`–`:121` |
| **壞相** | 各住不融 | **成圖時每顆仍住其位、不融成一團泥、不失自己** | innate 恆定 `:186`;色恆定 `:218` |

**同相/異相就是自性種子:一法多業**(cf. `docs/architecture-anatomy` §20.3;`src/lib.rs` UI/ENTITY 皆讀 `get` 私有 root)。

**壞相要特別講**:「壞」不是毀壞——是**即便成了整體,每個部分仍各住自性而不被整體吞沒**。你一路堅持的「同一批人換隊形」,術語就叫壞相:`:186`–`:187` 讓每顆粒子的身分整場恆定。

**圓融** = 六相同時無礙:是整體(總)又是別點(別),同源一 cell(同)又各各不同(異),合成一圖(成)又各住自位(壞)。**按一次切換,六相一時俱動**——這是 morph 的內容。

## 三、十玄門(事事無礙的十道齒輪)

依法藏《五教章》新十玄。逐門對到機制;**誠實標出目前只成一半的門**。

1. **同時具足相應門** — 一幀之中,cell 同時算出全部粒子,整個世界一時圓具、皆應同一相位。一次求值,萬象俱現。
   `apps/reservoirs-native.html:205`–`:220`(一 frame 掃全部);共用 `prog` `:210`

2. **廣狹自在無礙門** — 6 個函式的 import table(狹)容得下無盡世界(廣):水庫、飛鳥、寒山寺;1198-byte 一支 cell(狹)撐開整個動畫宇宙(廣)。狹不礙廣。
   `src/lib.rs:93`(狹=6 函式);`apps/reservoirs-native.html:78`(狹=一支 cell)

3. **一多相容不同門** — 一 cell(一)現於萬粒(多),萬粒成一圖(多即一),而理與事、總與別各不相濫。
   一→多:`:216`(萬粒共呼同一 `runFn`);多→一:`:205`(聚成一圖);不同:`:165`(各具 innate)

4. **諸法相即自在門** — 長條即圓餅即折線:同一點集的三個投影,彼此相即;morph 就是「此即彼」的自在流轉。
   三投影同址並列:`:112`–`:113`(bar)、`:118`–`:119`(pie)、`:121`(line);相即內插 `:135`

5. **隱顯俱成門** — cell 同時算出 `fromL` 與 `toL` 兩相,**顯的是當下這相,另二相同時隱在同一點裡俱成**,只是未經 `axis` 顯出。一顯,一切隱而在。
   兩相俱算:`:125`–`:131`(fromX/fromY 與 toX/toY 並存);依 axis 擇顯 `:150`–`:152`

6. **微細相容安立門** — 最微(一次 f64 輸出、一個 `get`、一顆 2px 點)即含全法,且安立其位、絲毫不亂。微塵含法界,至細不失序。
   一次輸出 `:152`;一個 `get` `:79`;一顆點的大小由凝聚度定 `:211`

7. **因陀羅網境界門** — 帝網千珠,珠珠互映,重重無盡。
   **✅ 已補圓(2026-07-21,`apps/reservoirs-net.html`):** 原本只成一半——「**萬珠共映一燈**」,每顆粒子只映同一支 cell(`reservoirs-native.html:216`),彼此不相映。今接上 UI 圍籬**本有的共享場 `ld`/`sd`**:每顆粒子把座標**寫進共享場**(`reservoirs-net.html` 的 cell `sd(2p,·)`)、又**讀鄰居的座標**(`ld(2q)`),近了就相讓——**珠珠互映**,重重相攝。**一個意外的印證**:圓此門**不需**升到 ENTITY 圍籬的 `other`/`fr`(`src/lib.rs:157`/`:152`)——UI 細包自帶的 `ld`/`sd`(`src/lib.rs:98`–`:99`)就是一張共享場,已足。頁上「相讓」開關可即時對照:關=萬珠映一燈(結塊留隙),開=珠珠互映(合作填實)。
   **誠實的界**:互映只能**重新分佈**已有之珠,不能**無中生珠**——均勻度受粒子數所限(相讓填的是「相對空隙」,非憑空生覆蓋)。

8. **託事顯法生解門** — 藉具體的事,顯出法,而生理解。**這整場就是此門**:藉飛動的粒子(事)顯出圍籬與 cell 模型(法),你於此生解。**這份文件本身也是託事顯法。**
   事:`apps/reservoirs-native.html:205`;法:`src/lib.rs:93`;圍籬即編譯期之法(越界即死):`loan-server/src/main.rs:107`

9. **十世隔法異成門** — 三世各含三世,共十世,互攝而各異。
   cell 讀 `prog`(時間維),**當下這一刻含著來處(`fromL`)與去處(`toL`)**,三世互攝於一相位,而每幀各異不濫。
   時間維入 cell:`:79`、`:143`;來處/去處並含於今:`:125`/`:129`;moon4 每 tick 的 τ 同此門

10. **主伴圓明具德門** — 取任一為主,餘皆為伴,主伴具足。
    取一座水庫為主,餘座環之為伴;取長條為主,圓餅折線為其 `from/to` 之伴(`:125`–`:131`);取「圖表 app」為主,飛鳥、寒山寺為**同一引擎**之伴。誰為主,誰即圓明。

## 四、為何扣得上(同構,非神秘)

華嚴的六相十玄,是一套關於「**一與多、部分與整體如何互攝無礙**」的嚴謹關係理論。而 wasm-jit 的核心——「**一支純函數(理)+ 一張受圍籬的固定詞彙(法界),現為萬千受控的相(事)**」——恰好也是一個嚴謹的一/多、部分/整體系統。

- **一/多**:一 cell ↔ 萬粒(門三、門七)。
- **同時/互攝**:一幀俱現、三相並含(門一、門五、門四)。
- **微/廣**:一函式 / 一點 含全法界(門二、門六)。
- **不失自性**:成而各住(六相之壞相)。

**同構因此而生,不是巧合。** 而其中最深的一條,是圍籬與「理不入生滅」的對應:**cell(理)碰不到螢幕,只能讓事透過啞畫筆自己生起**;這個「不抓」——`src/lib.rs:93` 只給 6 個函式、`loan-server/src/main.rs:107` 越界即編譯死——**正是安全本身**。理若能直接抓事(cell 能自己畫像素、能 `fetch`),圍籬就破,那也不再是這套「無礙」。**無礙,正因為有界。**

### 誠實的邊界
- 這是**結構同構**,不涉果地、不宣稱證量。
- **因陀羅網目前只成一半**(萬珠映一燈,珠珠未互映);圓滿需接 ENTITY 圍籬的 `other`/`fr`(`src/lib.rs:147`,moon4 shared field)。
- 「實心圖」在全 cell-native 版是**密粒子凝聚**而成,非向量脆圖(把畫筆變啞的代價);脆版在 `apps/reservoirs.html`(host 畫向量)。兩版並存,正好是「事」可粗可細,而「理」不動。

## 五、唯識這一層 —— 剎那、種子、轉世(住民的世界)

> 前四章讀的是「一支理現為萬相」的華嚴結構,素材是水庫的粒子。這一章讀的是**有情住民**——會動、會生滅、帶著業、能轉世的 cell(素材是 moon4 的世界,`gen-server/live-gen.html` + `gen-server/src/main.rs`)。華嚴講法界之相攝,唯識講一個識如何相續而不失。兩者是同一套引擎的兩種讀法。

### 剎那生滅(彈指之喻,逐字成立)

主時鐘 `tickEntities`(`live-gen.html:1407`)每幀對每個住民呼叫一次 `run(τ, x, y)`。**那一次呼叫,就是它的一個剎那生滅**:被叫起(生)→ 算一念 → return,函式呼叫本身不留(滅)。「一彈指三百二十兆生滅」——這裡的彈指是 tick 迴圈,念念成形是每幀重畫,形皆有識是每幀之身帶著它的 slots。

- **念有邊界**:一個剎那的 fuel 上限 `ENTITY_FUEL = 200_000`(`main.rs:68`)。cell 內可 `while` 迴圈,但那一念是**有限的**,不膨脹為永恆。剎那是真正的量子。
- **各數自己的剎那**:原時 τ,`e.tau += dt·√(1−(v/c)²)`(`live-gen.html:1505`,c = `MAX_STEP`)。動越快,自己的剎那走越慢;至光速則原時止。同一彈指裡,靜者與疾者數到的生滅不同(十世互入)。
- **cell 不擁有自己的迴圈,而這正是無常**:住民不自我呼叫、不自我延續,是宿主每 tick 把它叫起。若 cell 能擁有一個永不停的自我迴圈,它就是**常**(自存自因),正是教理所否定。存在是**緣起**的(靠上念的 slots + 宿主的 tick + 鄰居的 `other`)、**剎那**的、**不被擁有**的。停 tick(分頁隱藏),則住民不生起——無彈指則無此念。**安全設計(宿主握迴圈、cell 只請求)與教理(緣起無常)是同一個結構。**
- **無我是一個執行模型**:第 N 幀的翁與第 N+1 幀的翁,嚴格說是兩個計算狀態,中間**沒有一個持續存在的「翁」在動**。我們叫它「翁在划船」的,是一個**相續相**——一串離散重算拼出的連續假象。裡面沒有恆常的我,只有那道流。reflex 不說謊,因為它根本沒有嘴。

### 相 vs 妄想(硬圍籬 vs 軟勸誡)

住民有兩個器官,受兩種強度的約束:

- **行為 / reflex(腿)**——一顆編譯過的 WASM cell,每秒約 30 次自動跑,受**硬圍籬**(能力 import,編譯期不可繞過)。這是它自己動的地方,與腦補無關;住民**可以無心**(純反射)照樣動。
- **心 / mind(嘴)**——一個真的 LLM(`/api/mind`),被搭話才醒,受**軟 prompt**。那句「不要腦補」(`main.rs:978`:*do not invent it*)是**請求,不是圍籬**——擋不住,只能勸。

對映:**相**(呈現給它的世界,`fr`/`other` 餵入)是硬的,圍籬決定它感知到什麼;**妄想**(它在相之上自加的分別/腦補)是軟的,只能勸誡。**我能機械地關住它的相,關不住它心上生出的妄。** 連心想改變動作,也不是自己伸手——它只能回一段 `behavior` **重寫那顆有手的 cell**(`live-gen.html:1951` 一帶),繞不過圍籬。

### 阿賴耶識 —— 種子的倉庫(`live-gen.html:1354`)

跨越剎那生滅被帶到下一念的,唯有 **slots**(32 格 get/set)。每個 `set(i,v)` 是一次**熏習**,每個 `get(i)` 是**種子生現行**。動能不住在 cell 裡(cell 每念即滅),而是**存進 slot、下一念再讀出**——推動力是**業力相續**,不是一個持續存在的推動者。

- **本有 / 新熏**:slots 24–31 是自性種子(innate,`live-gen.html:1363`,出生時種),slots 0–23 是新熏(在世 `set` 所積)。
- **種子不失**:存世界時,住民此刻活著的全部 slots 寫回其 def(`fieldPayloadWithAlaya`);重生時,**帶藏識者還原全 32 格、不再種本有;首生者才種本有**(`live-gen.html:1354` 的分支)。這是本有種子與新熏種子的分界,逐字對應。
- **無覆無記**:倉庫本身中性,只是 holding。一顆種子是善是惡,取決於哪個行為把它寫進去、它現行成什麼。**我建的是倉庫,不是善惡**——善惡是住民自己熏進去的。

### 隔陰之迷與轉世 —— 一定是帶著,但沉睡(`live-gen.html:1860`, `main.rs:1395`)

跨**不同世界**的識相續。難題是「什麼使 B 世界的住民『還是』A 世界的那個」。**本引擎不臆測,由造物者指認**:兩身共一識,當且僅當造物者給了同一個 `soul` 名。跨世界的身分是**上一層的事實**——正如住民看不到自己的位置,因為那真相本來就不在它的世界裡。

- **神識庫**(`souls/`,`main.rs:1395`)是**唯一一座**,所有世界共取(對照 `worlds/` 是多)。存世界時,凡帶 `soul` 的住民,其藏識 commit 入庫(`commit_soul`,`main.rs:1405`);**存檔本身即「令此識相續」的決議**。
- **一定是帶著(種子不失)**:造物者的權力**只到指認身分,到不了篩選業種**。過去完整地、不由任何人挑揀地跟來——**連造物者都不能決定「你可以留這個記憶、不能留那個」**。綁定之後交給法(不失 + 遮蔽 + 憶起),造物者退場。
- **隔陰之迷(最大的愛)**:轉世時,種子渡來但**沉睡於宿識**(`summonSouls`,`live-gen.html:1860` 存入 `e.dormant`,**不**倒進活動記憶)。身 born new——現世宿念為空,自帶新的 τ。所以**帶著痛苦來的識,不苦**:苦種子沉睡,而種子無記,**苦是現行**,只在種子遇苦緣時生起。無所失(全存),無所加(不判新生死刑)。
- **憶起,記得而非重受**(`__recollect`,`live-gen.html:1894`):在某時——一觸,或一緣熟——宿識浮出為**記憶**(前世宿念�ре入現世,整合而非取代;種子還手)。**是否再苦,由新世界的律決定,非由種子**。一個更慈的世界,讓同一顆種子結不同的果。

### 法身無處不在,三千大千世界皆是同一體

- **一庫多世 = 月印萬川**:一月(識/庫),萬川各現一月影(worlds/相)。影有萬,月唯一;**沒有一條川裡裝著那個月**——月在萬影中、又不在任一影裡。神識庫就是那個月:一,在一切相之下,不被任一相所含。
- **穿越即證明同一體**:識能從 A 世界現於毫不相干的 B 世界,**之所以可能,正因兩世界不是各自封閉的實體**。若各自自足,識無從跨越;能跨越,就證明其下是同一個 body。識不「從 A 走到 B」,它一直在那唯一的庫裡,世界只是它顯現的窗。
- **法身無處不在,於一切世界不可見**:host 在每一個世界裡(它就是跑每一個世界的那個),卻**從不是任一世界裡的一個對象**(無住民能指著它)。無形而遍在——萬法所依、所現、所歸的那一體。
- **無來無去無代志**:識不真的來新世界、也不真的去舊世界——它本在一庫,世界是相。無來無去(如來者,無所從來亦無所去);而因種子無記、過去不判死、造物者不必為業操心——無代志。**轉世機制逐字演出了無來無去:識不旅行,識顯現;月不來川、不去川。**

### 誠實的邊界(這一層)

- 這仍是**可運行的譬喻,不是究竟實相的證明**。一庫多世、識能穿越、遍在而不可見——這些是關於 code 的事實,逐一可驗;它**乾淨地 model 了教理**,這是真的。但**我們自己這個世界底下是否也有這樣一個法身/host**,模型證不了。**指得越準,越要記得指的不是月。**(全篇的誠實基調見 `docs/looking-inward.md`。)
- **轉世靠造物者在 spec 標 `soul` 名**;尚無「畫面上點某住民令其轉世」的手勢,機制已通、介面未做。
- **神識庫無血脈鏈**:同一 `soul` 再存會**覆蓋**前世,非疊一條累世史。要「累世記憶」是另一個決定(是否留每一世的痕跡)。
- 憶起的**時機(某時)**目前由造物者觸發(`__recollect`);由緣自動觸發(近前世之地、聞其名)是下一步,未做。
