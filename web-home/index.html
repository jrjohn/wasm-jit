02

The code

The seed pipeline & the one wall

How does a script run on the world? The runtime compiles it into a tiny WASM module and hands it to the browser engine (V8 / JSC / SpiderMonkey) to JIT. The same DSL runs three ways; the generated WASM ties the AOT Rust ceiling and ties hand-written JS, value bit-identical. The point isn't speed (that ties JS anyway) — it's that "fast + sandboxed + synchronous" all hold at once.

One wall, two doors. The home DSL is blocked at codegen; WASM compiled from any other language is blocked before instantiate by audit(). One security model, looking at one thing — a module's declared imports ⊆ the capabilities the host grants. That lets the language grow past what fits in a prompt while the capability fence does not move an inch.

03

Extensibility

Grow the vocabulary, never the reach

There are only four legal shapes to add. Each is an addition to the capability vocabulary; none is a new reach.

soul · entity   run(t, ex, ey) sincosgetsetfrmvunbindriseother
skin   run(px, py, s, t, nx, ny)  — can only draw, cannot touch the world sincoshuergbhsldiscringarcline
mind · reply — the fields one heartbeat may return saythoughtbehaviorintentbegetskinattrsremember

Adding a compile target = give it a set of PARAMS + IMPORTS and audit it against a grant list; adding a capability = add a host import, and it will only ever be one of four kinds — data (attrs), sensing (other / perception), appearance (skin), or bounded generation (beget). Language-agnosticism comes for free: any language that compiles to WASM can be a soul — the audit reads only bytes. AssemblyScript souls have already walked through this door.

06

How to understand it

The mental model — hold three things

One sentence

The import table is the module's entire world. A capability not placed in it is a boundary the module cannot reach past. The fence's force is independent of the generator's intelligence.

Three kinds of "real-time," 1000× apart

Generate (s) / compile (ms) / execute (µs). Treat them as one and you misjudge cost; hold them apart and the architecture is clear.

Indra's net: store the rule, not the net

The relations among N components aren't N² wires but "beholding one another through a shared principle." Store the rule; the net forms only at manifestation.

Substance & principle: a soft thought riding a hard clock

Underneath, one shared clock t (the fact); yet all of time can fold into a seed (the principle) — that folding stays host-side.

The Dharma-realm canvas — an honest footnote

A note on the metaphor
Store the rule, not the net; let all things behold one another through a shared light without ever touching — and that light, in the end, is the prediction you and the one you interact with are correcting together, right now.

This architecture borrows a Buddhist vocabulary — Indra's net, ālaya, six roots and six consciousnesses, beings each with their own mind. These are coordinates, not proofs: they help one see "collective karma," "store the cause not the fruit," "capabilities attenuate" faster — but the architecture's correctness stands on bytes and SQL, not on scripture. As a finger pointing at the moon, gladly; as a proof, held at the door.
